# CLAUDE.md — gl-domains-list

Repo purpose: publish a single plain-text domain list (`list.lst`) consumed by [GL.iNet routers](https://docs.gl-inet.com/router/en/4/tutorials/how_to_configure_domain_and_ip_filtering_rules_for_glinet_routers_via_an_online_text_file/) for VPN Policy / Parental Control.

## Hard rules

- **Never edit `list.lst` directly** — it is regenerated by `scripts/build.sh` (CI commits this file). Manual changes will be overwritten on the next scheduled run.
- **Add domains to `list.manual.lst`**, not to `list.lst`.
- **Add categories to `scripts/categories.txt`**, one per line. `#` comments allowed. Unknown categories are silently skipped, so it's safe to list aspirational ones.
- **Format constraints (GL.iNet)** — `list.lst` may contain only:
  - bare domains (`netflix.com` — wildcard for all subdomains), or
  - exact FQDNs (`www.netflix.com` — exact match), or
  - IPv4 addresses (`34.111.220.252`), or
  - IPv4 CIDR ranges (`34.0.0.0/16`).

  Forbidden in `list.lst`: protocols, paths, comments, blank lines, v2ray prefixes (`domain:`, `full:`, `keyword:`, `regexp:`), attribute suffixes (`@cn`, `@ads`), IPv6 (not supported by GL.iNet subscription mode).

## Pipeline (scripts/build.sh)

1. Download latest `geosite.dat` from [`runetfreedom/russia-v2ray-rules-dat`](https://github.com/runetfreedom/russia-v2ray-rules-dat) release (verifies sha256).
2. Extract each category from `scripts/categories.txt` via [`geoview`](https://github.com/snowie2000/geoview).
3. Normalize: drop `regexp:` / `keyword:` / `@*` lines, strip `domain:` / `full:` prefixes, lowercase.
4. Merge with `list.manual.lst`, `sort -u`.
5. Collapse subdomain redundancy: if `foo.com` ∈ set, remove `*.foo.com` (GL.iNet root match wildcards subdomains).
6. Write `list.lst`.

The pipeline runs daily at 04:30 UTC via `.github/workflows/update-list.yml` (also `workflow_dispatch`-able). It commits directly to `main` when `list.lst` changes.

## When making changes

| Want to | Do |
|---|---|
| Add a single domain permanently | Append to `list.manual.lst`, commit. |
| Add a whole service (e.g., spotify) | Add category name to `scripts/categories.txt`, commit. |
| Drop redundant entries from manual list | The collapse step handles this automatically — no action needed. |
| Verify locally before push | `bash scripts/build.sh` then `wc -l list.lst` + spot-check. |
| Rotate geoview version | Bump `GEOVIEW_VERSION` in `scripts/build.sh` and `.github/workflows/update-list.yml`. |

## Common pitfalls

- **Geoview omits the trailing newline** on each output file — `build.sh` uses `awk 1` (not `cat`) for concatenation. Don't replace it with `cat` or you'll get merged-line garbage.
- **Concatenated lines like `www.amd.comanthropic.com`** — symptom of the above; means concat lost a newline.
- **Categories silently missing** — geoview's `-strict=false` means unknown category names just get logged as `(skipped)` rather than failing CI. If a category you expect isn't pulled, check it exists in [v2fly/domain-list-community/data](https://github.com/v2fly/domain-list-community/tree/master/data).
- **Subdomain redundancy**: `googleapis.com` already wildcards `content-autofill.googleapis.com`, so adding subdomains to `list.manual.lst` when the parent is also present is wasted bytes — the collapse step removes them.
